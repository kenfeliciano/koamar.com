## Syntax Highlighting

When the MDX started finally generating, syntax highlighting stopped working. I thought this would be fairly simple but, as with everything else, many iterations!

- At one point, AI had me put `mdx.table` and `mdx.pre` in my components list instead of `table` and `pre` as I had always. Well, back to my way.
  All is great except, after a bunch of other fixes, I see that highlighting is not working.
- After many other attempts at a fix, AI and I (well AI) wrote a rehype plug-in. It converts the Meta from the HAST (yes?) to `data-meta` that I can see and work with.
- This works perfectly!

```javascript
// rehype-meta-to-data-meta.js
import { visit } from 'unist-util-visit'

export default function rehypeMetaToDataMeta() {
  return (tree) => {
    visit(tree, 'element', (node) => {
      if (node.tagName !== 'code') return

      const meta = node.data && node.data.meta
      if (!meta) return

      node.properties = node.properties || {}
      node.properties['data-meta'] = meta
    })
  }
}
```

Add this new plugin to the `rehypePlugins` list in `gatsby-config` and then update the `preToCodeBlock` function to look for `data-meta` instead of `meta`.

```javascript
// in gatsby-config.mjs
import rehypeMetaToDataMeta from './plugins/rehype-meta-to-data-meta.mjs'
// ...
{
  resolve: `gatsby-plugin-mdx`,
  options: {
	extensions: [`.md`, `.mdx`],
	mdxOptions: {
	  remarkPlugins: [remarkGfm],
	  rehypePlugins: [rehypeMetaToDataMeta],
	},
	gatsbyRemarkPlugins: [
	  {
		resolve: `gatsby-remark-images`,
		options: {
		  maxWidth: 1200,
		  quality: 90,
		  linkImagesToOriginal: true,
		},
	  },
	],
  },
},

// in gatsby-wrapper.js - a new preToCodeBlock function that works with MDX 2
function preToCodeBlockV2(preProps) {
  const child = preProps.children

  if (
    child &&
    child.props &&
    typeof child.props.children === 'string' &&
    child.props.className &&
    child.props.className.startsWith('language-')
  ) {
    const language = child.props.className.replace('language-', '')
    const metastring = child.props['data-meta']
    return {
      codeString: child.props.children.trim(),
      language,
      metastring,
      ...child.props,
    }
  }

  return null
}
```
